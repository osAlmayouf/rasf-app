# دليل استخدام Git — مشروع RASF

---

## أولاً: الإعداد الأولي (مرة واحدة فقط)

```bash
git config --global user.name "اسمك"
git config --global user.email "بريدك@rasf.sa"
```

---

## ثانياً: تحميل المشروع لأول مرة (للمتعاونين الجدد)

```bash
git clone https://github.com/osAlmayouf/rasf-app.git
cd rasf-app
npm install
```

ثم انسخ ملف البيئة:
```bash
copy .env.example .env
```
وعدّل القيم داخل `.env` بالمفاتيح الحقيقية.

---

## ثالثاً: سير العمل اليومي

### 1. قبل البدء — جلب آخر التحديثات
```bash
git pull
```
> دائماً ابدأ يومك بهذا الأمر لتجنب التعارضات.

### 2. أثناء العمل — عرض التغييرات
```bash
git status
```
يُظهر الملفات التي تم تعديلها.

```bash
git diff
```
يُظهر تفاصيل التغييرات سطراً بسطر.

### 3. إضافة الملفات للحفظ
```bash
# إضافة ملف معين
git add src/components/dashboard/Dashboard.jsx

# إضافة كل التغييرات دفعة واحدة
git add .
```

### 4. حفظ التغييرات (Commit)
```bash
git commit -m "وصف مختصر للتغيير"
```

**أمثلة على رسائل جيدة:**
```bash
git commit -m "إضافة صفحة التقارير المالية"
git commit -m "تصحيح حساب IRR في مشروع أسار"
git commit -m "تحديث ترجمات اللغة الإنجليزية"
```

### 5. رفع التغييرات إلى GitHub
```bash
git push
```

---

## رابعاً: العمل على فرع منفصل (Branch) — الطريقة الموصى بها

بدلاً من العمل مباشرة على `main`، أنشئ فرعاً لكل مهمة:

```bash
# إنشاء فرع جديد والانتقال إليه
git checkout -b اسم-الفرع

# مثال
git checkout -b تطوير-صفحة-التقارير
```

بعد الانتهاء من التغييرات:
```bash
git add .
git commit -m "إضافة صفحة التقارير"
git push origin اسم-الفرع
```

ثم اذهب إلى GitHub وافتح **Pull Request** لدمج الفرع مع `main`.

---

## خامساً: أوامر مفيدة أخرى

| الأمر | الوظيفة |
|---|---|
| `git log --oneline` | عرض سجل الحفظات بشكل مختصر |
| `git branch` | عرض الفروع المتاحة |
| `git checkout main` | الرجوع للفرع الرئيسي |
| `git stash` | إخفاء التغييرات مؤقتاً دون حفظها |
| `git stash pop` | استعادة التغييرات المخفية |
| `git pull --rebase` | جلب التحديثات مع تجنب رسائل الدمج الزائدة |

---

## سادساً: حل مشكلة التعارض (Conflict)

إذا ظهرت رسالة تعارض عند `git pull`:

1. افتح الملف المتعارض — ستجد علامات مثل:
```
<<<<<<< HEAD
التغيير الخاص بك
=======
التغيير القادم من GitHub
>>>>>>> main
```
2. احذف العلامات واختر النسخة الصحيحة (أو ادمج الاثنتين).
3. ثم:
```bash
git add .
git commit -m "حل تعارض في اسم الملف"
```

---

## سابعاً: ما يجب تجنبه

- لا تحذف أو تعدل ملف `.env` في GitHub — فيه مفاتيح سرية.
- لا ترفع مباشرة على `main` إذا كنت تعمل على ميزة كبيرة — استخدم فرعاً.
- لا تستخدم `git push --force` إلا إذا كنت متأكداً تماماً.
